# Этап 3: Логика ротации ключей

## Задачи

- [ ] Реализовать алгоритм выбора наименее использованного ключа
- [ ] Добавить весовые коэффициенты для приоритизации
- [ ] Реализовать проверку доступности ключа
- [ ] Добавить fallback механизмы

## Алгоритм выбора

1. Фильтрация доступных ключей
2. Расчет score для каждого ключа:
   - Score = (usage_weight * usage) + (time_weight * time_since_last_use)
3. Выбор ключа с минимальным score
4. Проверка валидности ключа
5. Возврат выбранного ключа

## Веса

- `usage_weight` - важность количества использований
- `time_weight` - важность времени с последнего использования
- `limit_weight` - важность текущих лимитов

## Fallback

Если выбранный ключ недоступен:
- Попытка использовать следующий по приоритету
- Повторная попытка через N секунд
- Логирование ошибки
