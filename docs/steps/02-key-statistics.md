# Этап 2: Сбор статистики ключей

## Задачи

- [ ] Реализовать модуль сбора статистики
- [ ] Определить источники данных о использовании ключей
- [ ] Создать структуру для хранения статистики
- [ ] Реализовать кэширование статистики

## Источники статистики

Возможные источники:
- Логи opencode-go API
- Метрики использования
- Локальный файл с историей использования
- API провайдера ключей

## Структура данных

```python
{
    "key_id": {
        "usage_count": int,
        "last_used": timestamp,
        "limits": {
            "requests_per_minute": int,
            "requests_per_day": int
        },
        "current_usage": {
            "minute": int,
            "day": int
        }
    }
}
```

## Алгоритм сбора

1. Загрузка текущей статистики из кэша
2. Обновление данных из источников
3. Расчет метрик использования
4. Сохранение в кэш
