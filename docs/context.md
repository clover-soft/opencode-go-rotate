# Контекст проекта opencode-go-rotate

## Проблема

Ассистенты используют ключи opencode-go для доступа к API. При использовании одного и того же ключа в нескольких файлах настроек происходит неравномерное распределение нагрузки, что может привести к исчерпанию лимитов на одном ключе, в то время как другие ключи остаются недогруженными.

## Решение

Скрипт анализирует статистику использования ключей и автоматически определяет ключ с наименьшей нагрузкой. Затем этот ключ прописывается в соответствующих конфигурационных файлах, чтобы ассистенты использовали наиболее доступный ресурс.

## Основные компоненты

1. **Сбор статистики** - анализ использования каждого ключа (количество запросов, лимиты, временные метки)
2. **Выбор ключа** - алгоритм определения наименее использованного ключа
3. **Обновление конфигураций** - запись выбранного ключа в файлы настроек
4. **Логирование** - отслеживание операций ротации для аудита

## Хранение ключей

Ключи opencode-go хранятся в переменных окружения с префиксом `OPENCODE_GO_KEY_`:
```
OPENCODE_GO_KEY_001=key1
OPENCODE_GO_KEY_002=key2
OPENCODE_GO_KEY_003=key3
```

## Конфигурация файлов

Список файлов для обновления хранится в `config.yaml` в формате YAML:
```yaml
config_files:
  - "/path/to/.env1"
  - "/path/to/config.yaml"
```

## Файлы настроек

Скрипт работает со следующими типами файлов:
- `.env` файлы
- YAML конфигурации
- JSON конфигурации

## Критерии выбора ключа

При выборе ключа учитываются:
- Количество использований за период
- Текущие лимиты
- Время последнего использования
- Приоритет ключа (если настроен)

## Интеграция

Скрипт должен запускаться:
- По расписанию (cron)
- Вручную при необходимости
- В качестве хука при изменении конфигураций
